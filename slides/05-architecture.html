System Architecture · 总体架构

项目结构与模块连接

设计思路
Viewer 作为受保护资源，Context + API 工厂拆为 5 子接口；
功能模块通过 module.js 声明式注册，低耦合独立开发；
后端按路由分发到 core / services / 专用算法包。
DOVis/ ├── frontend/ React 19 + Vite + Cesium 1.141 │ ├── src/components/ AppShell · ModuleLauncher · FloatingWindowLayer │ ├── src/modules/ IsoSurfaceModule · ProfileModule · HypoxiaModule · eofModule │ ├── src/engine/ CesiumEngine · createCesiumApi · CesiumTilesRenderer │ ├── src/api/ CesiumAPIProvider (React Context 解耦 Viewer) │ ├── src/hooks/ useIsoSurface · useProfile · useEOF · useHypoxia … │ └── src/config/module.js 注册即拓展 ├── backend/ FastAPI + uvicorn (port 5001) │ ├── main.py / api/routers/ 路由入口：isoExport · profile · hypoxia · eof │ ├── core/ xarray + SQLite 缓存层 │ ├── isoGeometry/ Marching Cubes (PyVista + ECEF) │ ├── isoExport/ GLB → B3DM → 3DTiles 导出 │ ├── services/eof_service.py eofs 库求解 + 模态/方差/高值区结果 │ ├── services/ 剖面 · 低氧边界 · 等值面业务逻辑 │ └── hypoxia/ rasterio 等值线 → GeoJSON 边界 ├── data/ do_predict.nc (ML重建 1980-2019) └── tiles/ 3DTiles 缓存目录（哈希命名）
四功能模块内部结构
IsoSurface ├── Module.jsx 入口+布局 ├── useIsoSurface 体数据请求 ├── IsoSurfaceRenderer WebGL Ray Marching ├── ControlPanel 参数面板 └── useCesiumTiles B3DM加载
Profile ├── Module.jsx 入口 ├── useProfile 点选/断面请求 ├── ProfileChart Canvas 2D 曲线 ├── SectionChart Canvas 热力图 └── ControlPanel 模式切换
Hypoxia ├── Module.jsx 入口 ├── useHypoxia 阈值/深度参数 ├── useHypoxiaGeojson边界GeoJSON └── ControlPanel 渲染控制
EOF ├── Module.jsx 入口 ├── useEOF 分解请求 ├── ControlPanel 参数配置 ├── ResultPanel Plotly可视化 └── eofExport.js CSV/JSON导出
连接：前端模块声明能力与窗口策略 → 启动器生成入口并注入公共上下文 → 模块 Hook 通过 fetch 调用 FastAPI → 后端路由分发到 core / services / 专用算法目录 → 结果以 JSON / GeoJSON / 3DTiles 回流，由前端模块自行渲染与释放资源
